# 会员中心背景视频加载失败兜底说明

## 问题现象

- 打开「会员中心」时，控制台出现背景视频加载失败（例如 `assets/background/1.mp4` 404/加载失败）。
- 页面背景未正常播放动态视频。

## 修复内容

- 前端在 `miniprogram/pages/index/index.js` 中新增失败视频源缓存：
  - 一旦某个背景视频 URL 加载失败，会记录到 `failedBackgroundVideos`；
  - 同一会话内后续不再尝试该失效视频；
  - 自动回退静态背景图，避免重复报错和反复闪烁。

## 部署步骤

1. 在微信开发者工具上传并发布小程序（本修复仅涉及前端代码，不需要部署云函数）。
2. 清理本地缓存后重新进入「会员中心」验证：
   - 首次失败后应自动回退静态图；
   - 同一会话再次进入不应继续请求同一失效 mp4；
   - 页面可稳定展示静态背景。

## 验收建议

- 在 Network 面板确认失效视频仅触发一次失败请求（或显著减少重复请求）。
- 页面不再因视频失败出现大面积黑底或闪烁。
